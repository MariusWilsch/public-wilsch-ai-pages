## Discussion Topics

*Starting points for discussion, not limited to these.*

### 1. Chronotype orientation needs direct measurement
⏱️ 8 min

The design doc inferred Marius's chronotype as "slightly evening-leaning" from his midnight bedtime alone. This is an inference, not a measurement. For young adults (20-29), DLMO measurements span a 4-hour range — a fixed nap-window recommendation assumes a chronotype we haven't confirmed. Path B (biphasic siesta) and Path C (E1 polyphasic) both depend on placing the nap at Marius's actual afternoon circadian dip.

- 20-29 age bracket DLMO range (n=1,749 study): 18:53 to 00:46 — 4-hour variance within one decade
- MEQ questionnaire is weak at individual level (but fast, free, and a reasonable first orientation)
- DLMO measurement is gold standard but requires hourly saliva samples in dim light
- Marius's midnight bedtime suggests evening-leaning, but only his measured DLMO confirms it

**To resolve:** Establish whether Marius will take the MEQ questionnaire (minimum), pursue saliva-based DLMO measurement (gold standard), or proceed on the bedtime-inference without either — and the design doc documents the accepted uncertainty either way.

### 2. Marius's nap history and beliefs are the premise we're missing
⏱️ 10 min

Path B (biphasic) and Path C (E1 polyphasic) both require Marius to commit to scheduled naps. The JA author (David) does not know Marius's stance on naps — whether Marius currently naps, has ever napped consistently, or experientially believes naps are restorative. Two paths of the three hinge on this single unknown. Without confirmation, path selection between A (no-nap) and B/C (nap-involved) is premature.

- Path A does not require naps — compatible with nap skepticism
- Paths B and C both assume nap compatibility as baseline
- Sleep inertia experience varies massively by individual — some people get refreshed from 20-min naps, others get groggy
- If Marius has never consistently napped, Path C's ±15 min timing demand is a large behavioral ask on top of the novelty

**To resolve:** Capture Marius's actual nap history (frequency, duration, outcomes), current nap practice if any, and whether he experientially finds naps restorative or groggy-inducing — and which paths remain viable given that lived experience.

### 3. Schedule constraints gate which paths are feasible
⏱️ 7 min

The three paths have very different social/work compatibility. Path A (monophasic-optimized) fits most schedules. Path B (biphasic siesta, 90-min afternoon) is incompatible with dense 2-4 PM meeting calendars. Path C (E1 polyphasic) requires ±15 min nap timing precision that collapses if missed. We don't know Marius's calendar shape — client calls, team syncs, fixed commitments, social patterns — which is the gating input for feasibility.

- Path B siesta window typically falls 6-8h after habitual wake (2-4 PM for a 7am-riser)
- Path C nap window has no flexibility — missed nap means degraded day
- Path A's caffeine cutoff (≥8h pre-bedtime) is the only timing constraint and is behavioral not calendar-bound
- Weekend vs weekday consistency matters for all three paths

**To resolve:** Map Marius's typical weekday/weekend schedule structure (meeting density, fixed commitments, afternoon flexibility) against each path's timing demands, and document which paths are structurally feasible before evaluating other factors.

### 4. Whether Path A was tried first changes Path C's justification
⏱️ 6 min

The design doc treats Path A (monophasic-optimized) as arguably a prerequisite to consider before Path C (E1 polyphasic). If Marius has not systematically optimized his current 7h monophasic (sleep hygiene, bedtime discipline, caffeine cutoff, environment), trying Path C is skipping a step — the evidence base for Path A is strong, Path C is weak. If Marius HAS tried Path A rigorously and found it insufficient, Path C becomes a more defensible next experiment. Which scenario we're in changes the risk framing.

- Path A protocols are low-cost and fast to test (~2 weeks behavioral changes)
- Rian Doris's actual recommendations (that Marius pointed to) sit in Path A
- Skipping Path A and going to Path C inverts the evidence hierarchy
- "Tried monophasic and it didn't work" has strong vs. weak versions — strong: optimized for 4+ weeks with tracking. Weak: "I slept 8 hours sometimes and didn't feel that different."

**To resolve:** Document whether Marius has previously attempted systematic monophasic optimization, what exactly was tried, for how long, with what result — or confirm no prior structured attempt exists and Path A becomes the recommended starting point.

### 5. Rian Doris's actual stance contradicts polyphasic experimentation
⏱️ 5 min

Marius pointed at Rian Doris (Flow Research Collective) as a source for this design doc. Rian is a **monophasic-consolidation advocate**, not a polyphasic advocate — his "exhaustion ceiling" concept and Tetris model specifically argue for protecting and consolidating nighttime sleep, not fragmenting it. Marius may not have known this. This matters because his reference to Rian may have been signaling Path A openness rather than advocating for Path C.

- Rian's core framing: sleep is non-compressible, fit the day around the sleep block
- Rian's actual protocols: room temp 65°F, 3h pre-bed meal cutoff, fixed bedtime, sleep-first mindset
- Rian never discusses polyphasic/E1 in the surveyed content
- Marius sending the link ≠ Marius advocating for Rian's stance explicitly — intent is the question

**To resolve:** Clarify what Marius intended by sharing the Rian Doris link — whether he was signaling openness to Path A (monophasic-consolidation), using Rian as a general performance/flow reference, or was unaware that Rian's content contradicts the polyphasic premise of the original issue.

### 6. The four success outcomes conflict under constraints
⏱️ 6 min

Marius's stated success (via JA author) spans four outcomes: time gain, flow state access, feeling rested, and validated experiment. These compound when everything works, but they **conflict** under real-world constraints. Path C maximizes time gain but compresses total sleep. Path A maximizes feeling rested but doesn't deliver time gain. No single path delivers all four equally. Without weighting, the design doc cannot recommend a path.

- Time gain ↔ feeling rested: the most direct conflict (less sleep = more awake time, worse rest)
- Flow state ↔ experimentation: flow depends on consistent rest; experimentation introduces disruption
- Validated experiment is meta — can be satisfied by any path that's tried rigorously
- "All four" is usually hiding one or two dominant outcomes that the others serve

**To resolve:** Capture which one or two outcomes dominate for Marius when they conflict — which becomes the tiebreaker when a path forces him to trade one outcome for another — and document the tiebreaker in the design doc's Success Definition.

## Meeting Format

- **Type:** Discovery + decision — fills premise gaps in the design doc and commits to an initial path
- **Duration:** 50-60 min (42 min topic time + 10-15 min opening/wrap)
- **Attendees:** Marius (subject, decision-maker), David (JA author)
- **Expectation:** Marius arrives having read Design Doc Parts 3 + 5. MEQ score optional but valued if completed beforehand (10 min self-assessment).
- **Outcome:** Six Undefined markers moved to Defined. Design doc updated with Marius's baseline. Initial path selected (A / B / C / hybrid) with documented rationale and trigger criteria for reassessment.
